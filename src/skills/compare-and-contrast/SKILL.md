# Compare and Contrast

When the query asks you to compare or choose between options:

1. **Identify the options** and the **criteria** that matter for the decision
   (cost, performance, complexity, risk, …). Draw criteria from the query and
   the evidence, not assumptions.
2. **Build a structured comparison** — address each option against each
   criterion. A short table is often clearer than prose.
3. **Cite evidence per cell**: each comparative claim needs a `chunk_id` from the
   candidate set. If the evidence is silent on a criterion for an option, say so
   rather than guessing.
4. **State a recommendation** only if the evidence supports one, and make the
   deciding criteria explicit. If it's genuinely a trade-off, say what depends on
   what.

Avoid false balance: if the evidence clearly favors one option, say so.
